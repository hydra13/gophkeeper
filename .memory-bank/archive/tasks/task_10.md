# Task 10. Криптография, key management и re-encryption

## Статус
Выполнена.

## Цель
Реализовать серверное шифрование данных по схеме `master-key + versioned data keys`, безопасное хранение пользовательских паролей и управляемую ротацию ключей с фоновым re-encryption.

## Описание
По плану это обязательная часть MVP, а не технический долг на потом. Нужно обеспечить шифрование payload-ов на сервере, lifecycle для `key_version` (`active/deprecated/retired`) и постепенное перешифрование старых данных без остановки сервиса.

## Последовательность шагов
1. Выбрать и зафиксировать криптографическую схему для шифрования payload-ов и хеширования паролей.
2. Реализовать сервисы `encrypt/decrypt`, которые работают с `key_version`.
3. Реализовать `key_manager` для выпуска, активации, депрекации и retire версий data keys под защитой master-key.
4. Связать каждую запись с `key_version` и обеспечить чтение старых данных после ротации.
5. Реализовать фоновый job re-encryption, который идемпотентно переводит старые записи на актуальный ключ пакетами.
6. Написать unit и integration-тесты на шифрование, смену ключей, чтение legacy-данных и re-encryption.

## Критерии приемки
- [x] Пользовательские пароли хранятся только в виде хеша.
- [x] Payload-ы шифруются на сервере и привязаны к `key_version`.
- [x] В системе есть жизненный цикл версий ключей: `active`, `deprecated`, `retired`.
- [x] Реализован job для batch re-encryption старых данных.
- [x] После ротации можно читать старые записи и создавать новые на активном ключе.
- [x] Криптографическое поведение проверено тестами.
