# Task 22. Подтвердить и обеспечить покрытие тестами не ниже 70% по всей системе

## Статус
Выполнено.

## Цель
Сделать требование по покрытию прозрачным, воспроизводимым и проверяемым: проект должен действительно достигать порога не ниже 70% по всей системе или явно и честно пересмотреть формулировку в документации и quality gate.

## Описание
Сейчас проект декларирует выполнение требования по покрытию, но доступные артефакты ему противоречат, а quality gate в `Makefile` считает метрику не по всей системе. Для release-процесса это создаёт ложное ощущение готовности. Задача должна привести к одному из корректных результатов: либо проект реально достигает 70% общего покрытия и автоматом это доказывает, либо документация/quality gate перестают утверждать то, что не подтверждается. Предпочтительный путь для этой задачи: добиться фактического покрытия не ниже 70% по всему репозиторию MVP-кода и зафиксировать это в CI и документации.

## Последовательность шагов
1. Зафиксировать единое определение метрики покрытия для проекта: какие пакеты входят в “код всей системы”, какие исключения допустимы и почему.
2. Пересобрать пайплайн подсчёта покрытия так, чтобы он соответствовал зафиксированному определению и был воспроизводим локально и в CI.
3. Найти недопокрытые критичные области и добавить недостающие unit/integration/e2e-тесты прежде всего в high-risk путях.
4. Обновить `Makefile`, CI-конфигурацию и вспомогательные скрипты, чтобы порог 70% проверялся автоматически и прозрачно.
5. Обновить документы release readiness и quality gates по фактической метрике.
6. Сохранить артефакты/инструкции так, чтобы повторная валидация могла быстро подтвердить итоговую цифру.

## Критерии приемки
- [x] В проекте зафиксировано однозначное определение, как считается покрытие “по всей системе”, и это определение совпадает между кодом, CI и документацией.
- [x] Команда локальной проверки покрытия воспроизводима и даёт ту же логику подсчёта, что и CI.
- [x] Автоматический quality gate падает, если общее покрытие опускается ниже 70%, и проходит, если порог выполнен.
- [x] Итоговое покрытие действительно достигает не менее 70% по зафиксированной метрике, а не только после скрытых исключений критичных пакетов.
- [x] Недостающие тесты добавлены в проблемные области, которые раньше мешали достичь порога.
- [x] `docs/release-readiness.md`, README и другие документы не содержат утверждений о покрытии, которые нельзя подтвердить автоматической проверкой.
- [x] В репозитории есть понятный способ для ревьюера или валидатора быстро получить итоговую цифру покрытия и сверить её с порогом.
