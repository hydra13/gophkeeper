# Task 18. Документация и выпуск MVP

## Статус
Выполнено. Повторная валидация от 2026-03-30 пройдена, все критерии приёмки выполнены.

## Цель
Подготовить проект к выпуску MVP: актуализировать документацию, описать эксплуатационные процедуры и зафиксировать фактические границы первой поставки.

## Описание
Даже при готовом коде проект не соответствует исходному ТЗ без документации экспортируемых сущностей, описания запуска и явной фиксации scope. Эта задача завершает backlog и делает результат пригодным для передачи, сборки и дальнейшего сопровождения.

## Последовательность шагов
1. Обновить `README.md` и архитектурную документацию под фактический MVP scope: `server + cli`, `gRPC + HTTP`, `TLS only`, sync, chunk uploads, server-side encryption.
2. Описать запуск локального окружения, применение миграций, dev TLS-конфигурацию и базовый smoke-сценарий.
3. Подготовить эксплуатационные документы по key rotation, re-encryption и обработке инцидентов, связанных с ключами.
4. Проверить, что экспортируемые пакеты, типы и функции снабжены документацией.
5. Явно зафиксировать, что `desktop`, `web`, OTP, TUI и import/export находятся вне MVP и уходят в backlog.
6. Провести финальный release readiness review по чеклисту качества, безопасности и документации.

## Критерии приемки
- [x] Документация отражает реальный scope MVP и не обещает post-MVP функциональность как уже готовую.
- [x] Описаны локальный запуск, миграции, TLS-конфигурация и базовый smoke-сценарий.
- [x] Экспортируемые пакеты и типы снабжены документацией.
- [x] Есть эксплуатационные инструкции по key rotation и re-encryption.
- [x] Явно зафиксировано, что входит в MVP, а что вынесено в backlog.
- [x] Выполнен финальный review готовности к выпуску.
