# Название
Добавить валидацию ввода данных карт

# Статус: DONE

# Описание
При вводе данных карты нет валидации на номер карты, на экспайри, на владельце карты и на CVV. Получается, в эти данные можно вводить абсолютно любые значения, хотя номер карты он не может быть чем-то кроме числа. Ровно так же, как и с полем экспайри, туда можно ввести любой текст, что не является нормой. Причем эта валидация по сути должна быть на стороне бэкэнда. Но можно также еще и продублировать на стороне клиента.

# Решение

## Что сделано
Добавлена валидация `CardPayload` на уровне доменной модели с вызовом во всех транспортных слоях (HTTP POST/PUT, gRPC Create/Update).

## Правила валидации
| Поле | Правило |
|------|---------|
| Number | 13-19 цифр + алгоритм Луна |
| HolderName | непустое значение |
| ExpiryDate | формат MM/YY, месяц 01-12 |
| CVV | 3 или 4 цифры |

## Принцип
Валидация реализована на уровне доменной модели (`CardPayload.Validate()`), а не на транспортном слое. Это гарантирует единообразную проверку данных независимо от источника (HTTP, gRPC, будущие клиенты).
