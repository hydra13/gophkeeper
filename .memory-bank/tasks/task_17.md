# Task 17. Тестирование и CI

## Цель
Довести проект до инженерно устойчивого состояния: покрыть критичные части unit/integration/e2e тестами, достичь целевого покрытия и автоматизировать проверки в CI.

## Описание
ТЗ требует покрытие не менее 70%. Кроме покрытия нужны стабильные автоматические проверки для transport-контрактов, миграций, sync, uploads, TLS и CLI smoke-сценариев. Эта задача не меняет продуктовый scope, а закрепляет качество реализованного MVP.

## Последовательность шагов
1. Подготовить test matrix для домена, auth, CRUD, uploads, sync, TLS, миграций, key rotation и CLI.
2. Добрать недостающее покрытие unit, integration и e2e тестами до целевого порога не менее 70%.
3. Настроить CI pipeline с проверками `fmt`, `lint`, `test`, `cover`, генерацией proto и smoke-сборкой CLI.
4. Добавить автоматические проверки, что gRPC и HTTP transport-слои не расходятся по обязательным сценариям.
5. Убедиться, что post-MVP заготовки `desktop` и `web` не участвуют в основном CI как готовые продукты.

## Критерии приемки
- [ ] Общее покрытие проекта соответствует требованию не менее 70%.
- [ ] В CI автоматически запускаются линтеры, тесты, проверка покрытия и генерация контрактов.
- [ ] Есть test matrix, покрывающая домен, auth, CRUD, uploads, sync, TLS и key rotation.
- [ ] Есть smoke-проверка сборки CLI.
- [ ] Post-MVP заготовки не создают ложных CI-обязательств как готовые клиенты.
