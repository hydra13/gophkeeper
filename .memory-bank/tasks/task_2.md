# Task 2. Инженерная обвязка и базовые команды проекта

## Цель
Подготовить инженерную базу проекта: команды сборки, линтинга, тестирования, генерации контрактов и пример конфигурации, чтобы дальнейшие задачи выполнялись в едином стандарте.

## Описание
В ТЗ есть требования к качеству, покрытию и документации. Для передачи задач другой модели важно заранее закрепить единые команды, файлы конфигурации и правила запуска, иначе реализация расползется по стилю и механике. Эта задача не строит бизнес-функциональность, а создает рабочий контур разработки.

## Последовательность шагов
1. Подготовить или обновить `Makefile` с командами минимум для `fmt`, `lint`, `test`, `cover`, `proto`, `build`.
2. Настроить `.golangci.yml`, `.gitignore` и базовые правила для generated files.
3. Добавить шаблоны конфигурации в `configs/` и описать ожидаемые env/flags.
4. Зафиксировать единый способ локального запуска проверок и сборки проекта.
5. Проверить, что инженерные команды работают на уровне каркаса и не зависят от ручных неописанных действий.

## Критерии приемки
- [ ] В проекте есть единый набор команд для форматирования, линтинга, тестов, покрытия, генерации и сборки.
- [ ] Конфигурационные шаблоны лежат в `configs/` и соответствуют текущему MVP scope.
- [ ] Правила работы с generated files и временными артефактами зафиксированы.
- [ ] Базовый README или developer docs описывает, как локально запускать основные команды.
- [ ] Инженерная обвязка не противоречит плану `server + cli`, `gRPC + HTTP`, `TLS only`.
