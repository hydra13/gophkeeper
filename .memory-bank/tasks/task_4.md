# Task 4. gRPC контракты в `rpc/proto/v1`

## Цель
Спроектировать и зафиксировать versioned gRPC-контракты для auth, records, sync, uploads и служебных операций в `rpc/proto/v1`.

## Описание
По плану `gRPC` и `HTTP REST` являются равноправными transport-слоями. Эта задача отвечает только за gRPC-контракты, правила версионирования, общие DTO и генерацию protobuf-кода. Реализация серверной логики и HTTP-ручек сюда не входит.

## Последовательность шагов
1. Создать структуру `rpc/proto/v1` и разнести контракты по файлам `auth.proto`, `data.proto`, `sync.proto`, `uploads.proto`.
2. Описать сообщения и RPC для регистрации, логина, refresh/logout, CRUD записей, sync, upload/download и health-check.
3. Отразить в контрактах `device_id`, ревизии, soft delete, конфликты, `key_version`, chunk upload/download и resume.
4. Вынести shared messages и enum-ы в общие определения, если это устраняет дублирование.
5. Настроить генерацию protobuf-кода через стандартную инженерную команду проекта.
6. Добавить smoke-проверку, что proto-контракты компилируются и согласованы с layout проекта.

## Критерии приемки
- [ ] Все gRPC-контракты лежат в `rpc/proto/v1`.
- [ ] Контракты покрывают auth, records, sync, uploads и служебные методы.
- [ ] В контрактах отражены поля и сценарии, обязательные по плану MVP.
- [ ] Генерация protobuf встроена в инженерную обвязку проекта.
- [ ] Контракты документированы и готовы для server/client реализации.
