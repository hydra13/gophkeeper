# Task 13. Use-case слой uploads и бинарных вложений

## Цель
Реализовать общий use-case слой для бинарных вложений: создание upload session, прием чанков, завершение загрузки, скачивание и resume после обрыва.

## Описание
Бинарные данные входят в обязательный функционал проекта, но сценарии загрузки и скачивания существенно сложнее обычного CRUD. Эта задача вынесена отдельно, чтобы реализация была атомарной и не смешивала работу с binary attachments с базовыми операциями над записями.

## Последовательность шагов
1. Реализовать use-case слой для создания upload session и отслеживания состояния чанков.
2. Реализовать прием чанков, проверку порядка и допустимости операций, а также завершение загрузки.
3. Реализовать скачивание бинарных данных и сценарии resume после обрыва.
4. Подключить uploads use-case к HTTP upload endpoints и gRPC uploads service.
5. Согласовать error mapping для некорректного chunk sequence, незавершенной загрузки и отсутствующего вложения.
6. Покрыть unit и integration-тестами upload/download happy-path и resume/error сценарии.

## Критерии приемки
- [ ] Binary attachments загружаются и скачиваются чанками.
- [ ] Поддерживается resume для незавершенных upload/download операций.
- [ ] HTTP и gRPC используют один и тот же uploads use-case слой.
- [ ] Ошибки порядка чанков и незавершенной загрузки обрабатываются предсказуемо.
- [ ] Основные upload/download сценарии проверены тестами.
