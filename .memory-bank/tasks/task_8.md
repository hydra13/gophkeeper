# Task 8. Конфигурация, bootstrap сервера и TLS-only запуск

## Статус
Выполнена.

## Цель
Собрать инфраструктурный bootstrap сервера с единой конфигурацией, DI-проводкой зависимостей, запуском HTTP и gRPC transport-слоев и обязательным использованием TLS.

## Описание
После фиксации структуры и transport-контрактов нужен реальный каркас запуска, который связывает конфигурацию, логирование, transport-слои, middleware, auth-проверки и graceful shutdown. По плану любой сетевой доступ должен работать только через TLS. Эта задача готовит приложение к дальнейшему подключению persistence и use-case слоев, но не должна блокироваться их полной реализацией: wiring строится через интерфейсы и явные точки расширения.

## Последовательность шагов
1. Расширить конфигурацию под нужды MVP: адреса HTTP/gRPC, PostgreSQL, blob storage, JWT, TLS, crypto, upload limits.
2. Реализовать загрузку конфигурации из файла, env и флагов с валидацией обязательных параметров.
3. Собрать bootstrap приложения в `internal/app/server.go` с wiring для логгера, transport-слоев, middleware и интерфейсных зависимостей будущих repositories/services.
4. Поднять HTTP-сервер, gRPC-сервер и общий graceful shutdown, включая корректное завершение фоновых jobs и запуск хотя бы служебных endpoint-ов вроде health-check.
5. Добавить middleware и transport-level auth/rate-limit проверки для логирования, аутентификации и базовой защиты.
6. Подготовить dev-конфиг и dev TLS-сертификаты для локального старта без отключения шифрования.

## Критерии приемки
- [x] Сервер стартует из одной команды и поднимает одновременно HTTP и gRPC transport-слои.
- [x] Конфигурация валидируется до запуска приложения.
- [x] Все внешние зависимости подключаются через явный bootstrap и могут быть подменены до появления полной бизнес-реализации.
- [x] Запуск без TLS запрещен или завершается ошибкой конфигурации.
- [x] Реализован graceful shutdown для transport-слоев и фоновых jobs.
- [x] Есть пример конфигурации, достаточный для локального запуска MVP.
