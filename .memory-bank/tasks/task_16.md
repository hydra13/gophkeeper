# Task 16. CLI-клиент в `cmd/client/cli`

## Цель
Реализовать полнофункциональный CLI-клиент, покрывающий обязательные пользовательские сценарии GophKeeper и использующий общий client core.

## Описание
CLI является обязательным клиентом первой поставки. Он должен позволять пользователю зарегистрироваться, войти, синхронизировать данные и выполнять операции над секретами, не дублируя бизнес-логику и не обходя `pkg/client_core`.

## Последовательность шагов
1. Выбрать CLI framework или реализовать минималистичную собственную командную структуру.
2. Добавить команды `register`, `login`, `list`, `get`, `add`, `update`, `delete`, `sync`, `logout`.
3. Реализовать ввод чувствительных полей без эха и безопасную работу с бинарными файлами.
4. Добавить форматированный вывод записей, статусов синхронизации, ошибок конфликтов и технических ошибок.
5. Встроить вывод версии и даты сборки бинарника через `pkg/build_info`.
6. Написать e2e или smoke-тесты CLI на основные пользовательские сценарии.

## Критерии приемки
- [ ] CLI расположен в `cmd/client/cli`.
- [ ] CLI покрывает регистрацию, логин, CRUD, sync и logout.
- [ ] Команды используют shared client core и не дублируют бизнес-логику.
- [ ] Чувствительные значения вводятся без эха.
- [ ] Пользователь видит версию и дату сборки бинарника.
- [ ] Основные CLI-сценарии проверены e2e или smoke-тестами.
