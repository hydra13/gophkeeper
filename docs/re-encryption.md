# Перешифрование данных (re-encryption)

## Обзор

При ротации ключей шифрования часть данных остаётся зашифрованной устаревшими (deprecated) ключами. Фоновый job `reencrypt.Job` автоматически находит такие записи и перешифровывает их текущим активным ключом.

## Как работает

1. Job запускается с интервалом (по умолчанию 30 секунд)
2. Вызывает `keys.Manager.EnsureActive()` для получения текущего активного ключа
3. Запрашивает батч записей, у которых `key_version` не совпадает с активной версией
4. Для каждой записи:
   - Если тип `binary` — читает blob, расшифровывает старым ключом, шифрует новым, сохраняет
   - Если тип не `binary` — обновляет `key_version` на новый (payload уже перешифрован при обновлении записи)
   - Обновляет запись в БД с новым `key_version`
5. Цикл повторяется, пока есть записи для перешифрования

## Структура данных

Таблица `key_versions`:
| Колонка          | Тип       | Описание                            |
| ---------------- | --------- | ----------------------------------- |
| `id`             | BIGSERIAL | PK                                  |
| `version`        | BIGINT    | Порядковый номер версии (UNIQUE)    |
| `status`         | TEXT      | `active`, `deprecated` или `retired` |
| `encrypted_key`  | BYTEA     | Data key, зашифрованный master key  |
| `key_nonce`      | BYTEA     | Nonce для расшифровки data key      |
| `created_at`     | TIMESTAMPTZ | Время создания                     |
| `deprecated_at`  | TIMESTAMPTZ | Время депрекации (nullable)        |
| `retired_at`     | TIMESTAMPTZ | Время вывода из обращения (nullable)|

## Формат шифротекста

Шифротекст имеет формат: `GK1` + nonce (12 байт) + AES-256-GCM ciphertext. Префикс `GK1` позволяет отличить шифротекст от legacy-данных.

## Мониторинг

Для контроля состояния перешифрования:

1. Проверьте количество deprecated-ключей:
   ```sql
   SELECT version, status, deprecated_at FROM key_versions WHERE status = 'deprecated';
   ```

2. Проверьте количество записей, ожидающих перешифрования:
   ```sql
   SELECT COUNT(*) FROM records r
   WHERE r.key_version != (SELECT version FROM key_versions WHERE status = 'active' LIMIT 1);
   ```

3. После завершения перешифрования переведите deprecated-ключи в retired через `keys.Manager.Retire(version)`

## Устранение неполадок

| Проблема                       | Решение                                                |
| ------------------------------ | ------------------------------------------------------ |
| Job не обрабатывает записи     | Проверьте наличие active-ключа в `key_versions`        |
| Ошибка при расшифровке blob    | Убедитесь, что мастер-ключ в конфиге корректный        |
| Медленное перешифрование       | Увеличьте `batchSize` (через `WithBatchSize` опцию)    |
| Blob-файл не найден            | Проверьте путь в `blob.path` и целостность хранилища   |
